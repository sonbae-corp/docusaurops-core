## How DocusaurOps Makes This Real

<Tabs>
  <TabItem value="setup" label="🚀 Get started in minutes" default>

Install the GitHub Copilot plugin, tell the agent:

```
"Setup documentation for my project"
```

DocusaurOps handles the rest — scaffolding the Docusaurus site, deploying it to the shared platform, and wiring up the CI/CD pipeline. No hosting setup, no authentication configuration, no manual pipeline authoring.

  </TabItem>
  <TabItem value="search" label="🔎 Unified search">

Every site created with DocusaurOps is automatically crawled and indexed under a single URL. Your documentation gets a free, organization-wide search.

  </TabItem>
  <TabItem value="agent" label="🤖 Ask the agent">

All indexed content is made available through the **DocusaurOps agent**, directly in **Copilot CLI and VS Code**. Ask questions like:

- *"Where is the documentation for project X?"*
- *"What are the best practices for Y?"*
- *"Who owns the runbook for Z?"*

And get instant answers sourced from all content across the organization.

  </TabItem>
</Tabs>

:::note Scope & Limitations
| Topic | What to expect |
|---|---|
| **Permissions** | No access control system. Content published is assumed suitable for broad internal sharing. |
| **Content ownership** | Teams are responsible for their own content — ensure nothing confidential is published. |
| **Hosting scope** | Documentation sites only. DocusaurOps is not a general-purpose hosting platform. |
:::

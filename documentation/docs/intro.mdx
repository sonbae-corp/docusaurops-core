# Why DocusaurOps ?

## What problem we are trying to solve?

Currently, whithin organizations, project/technical documentation that could have a general interest for work teams is spread in multiple places using heterogeneous formats (ex: Word documents in obscur SharePoint libraries, web pages on standalone sites, PDFs, etc.) making it hard to read, difficult to access and sometimes completly hidden to teams. Users simply don't know where to start when looking for technical resources as there is no global structure. Project documenation slowly becomes a giant mess. Technical documentatioon is often underconsidered and seen as a burden for technical teams. It is often done at the end of projects, in a rush and with no real strategy, leading to poor quality content that is not maintained and quickly becomes obsolete.

Well executed, documentaion defines the "savoir-faire" in an enterprise. Without a proper strategy, knowledge is lost when peopl leave. It is a real problem for organizations as it leads to:

- Technical documentation scattered across multiple repositories, SharePoint sites, local drives, etc.
- Heterogeneous formats (Word, PDF, Markdown, etc.) making it hard to read and access
- Lack of discoverability and visibility of existing content across teams.
- No single reference point to access technical documentation across the organization.
- Bridge the gap between technical teams and other project stakeholders (ex: business analysts, security teams, etc.) on documentation and specifications.

**Without DocusaurOps**

{/* !["Before DocusaurOps"](./img/before_DocusaurOps.jpg) */}

The goal of the DocusaurOps platform is to provide an easy for technical teams to setup and managed their documentation without the hassle of hosting, deployment, authentication or complex configuration, while providing a single reference point for users to access technical content across the company using a friendly web format. 

It streamlines the approach to write beautiful technical documentation using a **'documentation-as-code'** approach leveraging Markdown format and the [Docusaurus static site generator](https://docusaurus.io/).

All sites created with DocusaurOps get their content indexed and available for search through the agent. You can ask it questions like "Where is the documentation about project X?" or "What are the best practices for Y?" and it will provide you with relevant information from all the content available on DocusaurOps.

## What benefits?

- 🎯 Improves teams efficiency by encouraging them to quickly create appealing technical documentation consumed as web format. No complex configurations, hosting setup or authentication. DocusaurOps is Ready to go!
- 🎯 Improves knowledge sharing and content discoverability across an organization via an unified search capability and a dedicated agent right in your work tool (Copilot CLI/Visual Studio code).
- 🎯 Provides a single reference point for users to access technical documentation across the company, improving visibility and accessibility of content. No more Word documents lost in a random SharePoint library. All you need is accessible easily with up to date content thanks to documentation as code.
- 🎯 Reduces the barrier to entry for teams to create and maintain technical documentation by providing a streamlined approach leveraging Markdown and Docusaurus. No need to worry about hosting, deployment, authentication or complex configuration. DocusaurOps handles all of that for you.

## What DocusaurOps doesn't include?

- ❌ **Permissions management system**: DocusaurOps does not provide any permisions management mechanism. If you wish to publish your content through the platform, we assume the content is suitable for internal usage as the gooal is to share knowledge across teams, not hide it from them.
- ❌ **Content ownership**: Teams are responsible for their own content and need to make sure it can be published and not disclose any private or confidential information.
- ❌ **Application hosting**: Except fron your documentation, we don't host any custom application. This is not a free hosting service for your own projects.

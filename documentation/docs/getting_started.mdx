## Prerequisites

Make sure you have all of these before getting started:

- DocusaurOps infrastructure [deployed](./deployment.mdx) and up and running
- [GitHub Copilot license](https://github.com/features/copilot/plans)
- [Visual Studio Code](https://code.visualstudio.com/)
- [Git](https://git-scm.com/)
- [GitHub CLI](https://cli.github.com/)
- [GitHub Copilot CLI](https://github.com/features/copilot/cli)

## Step 1: Install the plugin

<Tabs defaultValue="cli" values={[
{ label: 'Copilot CLI', value: 'cli' },
{ label: 'Copilot in VS Code', value: 'vscode' },
]}>
<TabItem value="cli">

**On your terminal:**

1. Add your GitHub marketplace:

```bash
copilot plugin marketplace add your-org/docusaurops-core
```

<div className="flex justify-center my-6">
    <img className="w-[80%]" src={CopilotCLIMarketplace}/>
</div>

2. Install the plugin:

```bash
copilot plugin install docusaurops@docusaurops-plugins
```

<div className="flex justify-center my-6">
    <img className="w-[80%]" src={CopilotCLIPluginInstall}/>
</div>

</TabItem>
<TabItem value="vscode">

**In Visual Studio Code:**

1. Open settings with `Ctrl+,`, search for `chat.plugins.marketplaces`, and add `your-org/docusaurops-core`:

<div className="flex justify-center my-6">
    <img className="w-[60%]" src={VSCodePluginMarketplace}/>
</div>

2. Open the Extensions panel, search `@agentPlugins docusaurops`, and click **Install**:

<div className="flex justify-center my-6">
    <img className="w-[60%]" src={VSCodeInstallPlugin}/>
</div>

</TabItem>
</Tabs>

## Step 2: Launch the agent

<Tabs defaultValue="cli" values={[
{ label: 'Copilot CLI', value: 'cli' },
{ label: 'Copilot in VS Code', value: 'vscode' },
]}>
<TabItem value="cli">

**Start the CLI and select DocusaurOps:**

1. Launch Copilot CLI:

```bash
copilot
```

2. Select the agent:

```bash
/agent
```

Then choose **DocusaurOps Agent**:

<div className="flex justify-center my-6">
    <img className="w-[80%]" src={CLISelectAgent}/>
</div>

3. Pick your AI model (optional but recommended):

```bash
/model claude-sonnet-4.6
```

:::tip Model Choice
We recommend **Claude Sonnet 4.6** for best results, but any available model works fine.
:::

</TabItem>
<TabItem value="vscode">

**Start a chat and confirm the agent is loaded:**

1. Open a new Copilot Chat window (`Ctrl+L` or via the Chat icon)

2. You should see the **DocusaurOps Agent** available:

<div className="flex justify-center my-6">
    <img className="w-[50%]" src={AgentChatVsCode}/>
</div>

3. Click on it or mention `@docusaurops` in your prompt and you're ready to go!

</TabItem>
</Tabs>

### Create & Deploy Documentation Sites

**docs-setup** scaffolds a complete documentation site, deploys it to Azure, and sets up automated CI/CD — all from a single prompt.

<div className="flex justify-center my-6">
  <div style={{ width: '100%', maxWidth: '560px', aspectRatio: '16/9' }}>
    <iframe src="https://www.youtube.com/embed/rcS7U7C5ufg" style={{width: '100%', height: '100%'}} allow="autoplay; encrypted-media" allowFullScreen />
  </div>
</div>

**Try this simple prompt:**

```
Setup a documentation site for project "HR Smart Assistant" with url "hr-smart-assistant"
```

**What happens automatically:**
- ✓ Downloads the latest template
- ✓ Customizes with your information
- ✓ Creates the documentation structure
- ✓ Deploys to Azure
- ✓ Sets up CI/CD pipeline
- ✓ Runs locally for you to test

After setup, your site will be live at: `https://<my-site>.<location>.cloudapp.azure.com/hr-smart-assistant`

After setup, the new site is deployed under the root website with the URL specified ex:( `https://<my-site>.<location>.cloudapp.azure.com/hr-smart-assistant`):

<div className="flex justify-center mb-10">
    <img className="w-[80%]" src={NewProjectSite}/>
</div>

When you'll commit and push to your repository, the documentation site will be automatically deployed to Azure through GitHub Actions, without requiring any setup on your repo.

### Search Across All Your Documentation

**docs-ask** lets you query your entire organization's documentation at once through natural language, finding answers across all indexed sites.

<div className="flex justify-center my-6">
  <div style={{ width: '100%', maxWidth: '560px', aspectRatio: '16/9' }}>
    <iframe src="https://www.youtube.com/embed/xGUISHrJpyo" style={{width: '100%', height: '100%'}} allow="autoplay; encrypted-media" allowFullScreen />
  </div>
</div>

**Try these prompts:**

**Find info about a specific project:**
```
Where is the documentation about project X?
```

**Find examples across your organization:**
```
Find me examples where we implemented feature XYZ and explain the architecture
```

**Get guidance on practices:**
```
What are the best practices for setting up authentication in microservices?
```

---

### Generate Specs from Requirements

**docs-specs** converts requirement documents into technical specifications and automatically creates GitHub issues for implementation.

<div className="flex justify-center my-6">
  <div style={{ width: '100%', maxWidth: '560px', aspectRatio: '16/9' }}>
    <iframe src="https://www.youtube.com/embed/cUkXb5Wmd3o" style={{width: '100%', height: '100%'}} allow="autoplay; encrypted-media" allowFullScreen />
  </div>
</div>

**Try this prompt:**

**Convert a requirement document:**
```
Generate technical specifications from this requirement document [link to SharePoint or OneDrive file]
```

The agent will:
- ✓ Read the requirement document
- ✓ Generate technical specifications
- ✓ Create GitHub issues for your team
- ✓ Structure work for implementation

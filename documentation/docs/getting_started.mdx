# Getting started with DocusaurOps plugin

DocusaurOps is a GitHub Copilot plugin for GitHub Copilot in VSCode and CLI focused on documentation management for technical teams within an organization.
It provides a set of agents and skills to help you in your documentation management across your organization's technical teams.

## Prerequisites
Here are the prerequisites you'll need to use the DocusaurOps plugin:

- DocusaurOps infrastructre [deployed](./deployment.mdx) and up and running.
- [GitHub Copilot license](https://github.com/features/copilot/plans)
- [Visual Studio Code](https://code.visualstudio.com/)
- [Git](https://git-scm.com/)
- [GitHub CLI](https://cli.github.com/)
- [GitHub Copilot CLI](https://github.com/features/copilot/cli)

## Use the agent

The **DocusaurOps Agent** is designed to assist you in creating and managing your documentation in your projects. To achieve this, it multiple following skills available:

#### `docs-setup`

Sets up a new documentation site by cloning the latest template from a dedicated repository, customizing it with your information, and configuring CI/CD pipelines to deploy your site on Azure. Also handles upgrading an existing documentation site to the latest template.

The site is published under a well known URL and then indexed through a Copilot connector (ex: `https://<my-site>.<location>.cloudapp.azure.com/`).

To use it, simply ask the following:

> Setup a new documentation site for the current project:

You: "Setup a documentation site for project "HR Smart Assistant" with url 'hr-smart-assistant'

> Create a site with demo content and authentication enabled:

You: "Setup a documentation site for project "HR Smart Assistant" with url 'hr-smart-assistant'. Use demo content and enable authentication".

`copilot --model claude-sonnet-4.6 --agent docusaurops:docusaurops --prompt "Setup a documentation site for project 'HR Smart Assistant' with url 'hr-smart-assistant'. Use demo content and enable authentication" --allow-all`

> Only `site_title` and `site_slug` are required.

**What happens at a high level:**
1. Downloads the latest DocusaurOps documentation template using your GitHub credentials (using `gh auth token`).
2. Fills in the provided information and prompts you to confirm or supply missing details.
3. Merges the template with your existing documentation, or creates the documentation structure if it doesn't exist.
4. Create demo content if requested.
5. Runs the documentation site locally.

After setup, the new site is deployed under the root website with the URL specified ex:( `https://<my-site>.<location>.cloudapp.azure.com/hr-smart-assistant`):

When you'll commit and psuh to your repository, the documentation site will be automatically deployed to Azure through GitHub Actions, without requiring any setup on your repo.

#### `docs-ask`

Asks questions about the content of all DocusaurOps sites using leveraging WorkIQ CLI MCP. Returns an answer based on relevant content found across all indexed sites.

**Usage:**

> Ask about a specific project:

You: "Where is the documentation about project X?"
You: "Explain the architecture on project Y based on the available documentation."

> Ask about a specific topic across all projects:

You: "Find me examples where we implemented a the feature XYZ and explain the architecture we used.""
You: "What projects are using these technologies already?" 
You: "Find guidance on how to setup feature XYZ"
